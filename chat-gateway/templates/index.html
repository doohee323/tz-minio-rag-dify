TZ-Chat Gateway

Unified chat API in front of Dify. Part of the tz-minio-rag-dify stack.

What this is

This service is the TZ-Chat Gateway: a single API and chat UI for multiple apps (e.g. DrillQuiz, CoinTutor). It sits in front of Dify, manages users and conversations, and can sync chat history to its own database for support and analytics. Sample integration: devops.drillquiz.com (DrillQuiz).

Project: tz-minio-rag-dify

The full stack runs on Kubernetes: MinIO (object store), Qdrant (vector DB), RAG backends (CoinTutor / DrillQuiz), Dify (chatbots), and this TZ-Chat Gateway. RAG ingestion runs on a schedule; Dify apps call the RAG backends as tools.

1. Workflow

Check out this repo, use sample apps (DrillQuiz / CoinTutor) as reference to register and apply your app.

Flow: add your app, then use chat
1. Check out this code → 2. Register your app (reference: DrillQuiz / CoinTutor) → 3. Get token & open chat page → 4. Chat
You add and register your app; you do not choose from existing apps. Token: issued after your app’s login, or call /v1/chat-token with API Key.

2. How the app operates

Request-to-response processing order

Request → response sequence
Client Send message from chat UI (JWT or API Key)
↓
TZ-Chat Gateway Validate JWT/API Key → sync conversation/messages to DB → proxy to Dify API
↓
Dify Run DrillQuiz or CoinTutor app (calls RAG tool when needed)
↓
RAG Qdrant + RAG Backend search → return results to Dify (required)
↓
Response Dify → TZ-Chat Gateway → client (streaming or normal response)

4. Apply to your project

Steps to add chat to an existing project (e.g. Vue app)

Integration order
1. Deploy TZ-Chat Gateway → 2. Configure env → 3. Backend token → 4. Frontend widget → 5. CORS & CSP
1. Deploy Run TZ-Chat Gateway on server (Docker/K8s or ./local.sh). Set JWT secret, Dify API key, CHAT_GATEWAY_API_KEY in .env.
2. Env CHAT_GATEWAY_JWT_SECRET, CHAT_GATEWAY_API_KEY, DIFY_*_API_KEY. See .env.example, scripts/gen-env-values.sh.
3. Backend Issue JWT for logged-in users from your backend, or have frontend call GET /v1/chat-token?system_id=...&user_id=... with X-API-Key.
4. Frontend Copy sample/ChatWidget.vue → wire authService (getUserSync, subscribe) → add <ChatWidget /> in App.vue. Set VUE_APP_CHAT_GATEWAY_URL, VUE_APP_CHAT_GATEWAY_API_KEY.
5. CORS·CSP Allow your project domain in TZ-Chat Gateway (ALLOWED_CHAT_TOKEN_ORIGINS). If the app has CSP, add TZ-Chat Gateway URL to connect-src and frame-src.
